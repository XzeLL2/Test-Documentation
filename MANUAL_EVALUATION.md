## 1. ВВЕДЕНИЕ

### 1.1 Назначение документа

Настоящий документ устанавливает порядок проведения ручной (экспертной) оценки качества изображения IP-видеокамер методом визуального осмотра без использования автоматизированных программных средств измерений. Документ предназначен для специалистов по тестированию видеооборудования, инженеров технического контроля, а также для использования в полевых условиях при отсутствии специализированного программного обеспечения.

### 1.2 Область применения

Методика применяется в следующих случаях:

- Первичная оценка качества изображения при вводе камеры в эксплуатацию
- Экспресс-анализ качества
- Сравнительная оценка нескольких камер без использования ПО
- Проверка работоспособности камеры при испрвлении ошибок
- Обучение персонала методам визуальной оценки
- Резервная методика при отказе автоматизированной системы

### 1.3 Отличие от автоматизированной оценки

Ручная экспертная оценка является качественной и основана на субъективном восприятии человека-оператора. Несмотря на меньшую точность, экспертная оценка позволяет выявить субъективно воспринимаемые дефекты, которые могут не фиксироваться автоматизированными системами, такие как:

- Утомляемость глаз при длительном наблюдении
- Субъективное восприятие «комфортности» изображения
- Визуальные артефакты, специфичные для определённых типов матриц
- Особенности цветопередачи, воспринимаемые человеческим глазом

### 1.4 Результат оценки

Результат оценки — **бинарный** (ГОДЕН / НЕ ГОДЕН). Для каждой характеристики принимается одно из двух решений:

- **ГОДЕН** — характеристика соответствует минимальным требованиям для решения задач наблюдения
- **НЕ ГОДЕН** — характеристика не соответствует минимальным требованиям

## 2. НОРМАТИВНЫЕ ССЫЛКИ И ТЕРМИНЫ

### 2.1 Термины и определения

| Термин | Определение |
|--------|-------------|
| **IP-видеокамера** | Цифровая видеокамера с сетевым интерфейсом, передающая видеоданные по протоколу IP |
| **Шум (Noise)** | Случайные флуктуации яркости/цвета на изображении, обусловленные физическими процессами в сенсоре и электронике камеры |
| **Контрастность (Contrast)** | Разность между максимальной и минимальной яркостью изображения, определяющая различимость объектов |
| **Резкость (Sharpness)** | Степень чёткости границ объектов и различимости мелких деталей |
| **Пересвеченность (Overexposure)** | Потеря деталей в ярких областях изображения вследствие избыточной экспозиции |
| **Цветопередача (Color Accuracy)** | Степень соответствия воспроизводимых цветов эталонным значениям |
| **SMPTE Color Bars** | Стандартная тестовая таблица для проверки цветопередачи (8 вертикальных полос) |
| **Экспертная оценка** | Субъективная оценка качества человеком на основе визуального анализа |

### 2.2 Регламинтирующие стандарты

- ГОСТ Р 51558-2014 «Средства и системы охранные телевизионные. Классификация. Общие
технические требования. Методы испытаний»
- Рекомендации SMPTE EG 1-1990 (для тестовых таблиц)

## 3. ПЕРЕЧЕНЬ ОЦЕНИВАЕМЫХ ХАРАКТЕРИСТИК

### 3.1 Перечень

Документ устанавливает оценку следующих пяти характеристик качества изображения:

| № | Характеристика | Английское наименование | Метод оценки |
|---|---------------|---------------------|--------------|
| 1 | Шумность | Noise | Визуальный анализ зернистости изображения |
| 2 | Контрастность | Contrast | Визуальный анализ диапазона яркости |
| 3 | Резкость | Sharpness | Визуальный анализ чёткости границ |
| 4 | Пересвеченность | Overexposed | Визуальный анализ ярких областей |
| 5 | Цветопередача | Color Accuracy | Визуальный анализ воспроизведения цветов |

### 3.2 Веса характеристик

Для справки, в автоматизированной системе используются следующие веса характеристик:

| Характеристика | Вес |
|---------------|-----|
| Резкость | 30% |
| Шумность | 20% |
| Контрастность | 20% |
| Цветопередача | 20% |
| Пересвеченность | 10% |

При ручной оценке все характеристики равнозначны, однако на практике наибольшее значение имеет резкость, так как она напрямую влияет на возможность идентификации объектов.

## 4. ПОДГОТОВКА К ПРОВЕДЕНИЮ ОЦЕНКИ

### 4.1 Требуемое оборудование

Для проведения ручной экспертной оценки требуется следующее оборудование:

#### 4.1.1 Основное оборудование

| Оборудование | Требования | Назначение |
|--------------|-------------|------------|
| IP-видеокамера | Объект оценки | Испытываемое оборудование |
| Монитор | Диагональ не менее 19", разрешение не менее 1280×1024 | Отображение видеопотока |
| Штатив или кронштейн | Стандартное крепление 1/4" | Фиксация камеры |

#### 4.1.2 Дополнительное оборудование (рекомендуется)

| Оборудование | Требования | Назначение |
|--------------|-------------|------------|
| Тестовая таблица SMPTE Color Bars | Формат А3 или больше, соответствие стандарту | Оценка цветопередачи |
| Источник освещения | Регулируемый, 300-500 люкс | Обеспечение требуемых условий |
| Люксометр | Диапазон 10-2000 люкс | Измерение освещённости |
| Ноутбук или ПК | Для подключения и настройки камеры | Настройка и мониторинг |

### 4.2 Требования к условиям проведения

#### 4.2.1 Освещённость

Оценка проводится при следующих уровнях освещённости:

| Уровень | Освещённость (люкс) | Условия |
|---------|---------------------|----------|
| Нормальный | 300-500 | Офисное освещение, дневной свет |
| Высокий | 1000+ | Прямой солнечный свет |
| Низкий | 50-100 | Сумерки, слабое освещение |

Примечание: Основные измерения проводятся при нормальном освещении (300-500 люкс).

#### 4.2.2 Температурный режим

Температура в помещении: 20-25°C (комнатная температура).

#### 4.2.3 Требования к монитору

Монитор должен быть предварительно откалиброван. Рекомендуется использовать следующие настройки:

- Яркость: 50-70%
- Контрастность: 70-80%
- Цветовая температура: 6500K (дневной свет)
- Режим отображения: sRGB

ВНИМАНИЕ: Некалиброванный монитор может искажать цвета и яркость, что повлияет на результаты оценки. Рекомендуется использовать монитор профессионального класса.

### 4.3 Подготовка испытательного стенда

#### Этап 1: Сборка оборудования

1. Разместите камеру на штативе или кронштейне
2. Направьте объектив камеры на тестовую таблицу SMPTE или на сцену с разнообразными объектами
3. Убедитесь, что тестовая таблица (или основные объекты) полностью помещаются в кадре
4. Подключите камеру к сети Ethernet
5. Подключите монитор к источнику видеосигнала (ПК, видеорегистратор)

#### Этап 2: Настройка освещения

1. Включите источник освещения
2. Установите освещённость в соответствии с требуемыми условиями (300-500 люкс)
3. При отсутствии люксометра оцените освещённость визуально: должно быть достаточно светло для чтения текста

#### Этап 3: Настройка камеры

1. Подключитесь к веб-интерфейсу камеры (или используйте специализированное ПО)
2. Установите следующие параметры:
   - Разрешение: 1920×1080 (Full HD) или максимальное
   - Частота кадров: 25-30 fps
   - Битрейт: 2048-4096 кбит/с
   - Кодек: H.264
3. Отключите функции, ухудшающие качество (шумоподавление, сжатие)
4. Отключите WDR (Wide Dynamic Range), если используется, для чистоты оценки

#### Этап 4: Прогрев камеры

1. Включите камеру и дайте ей прогреться в течение 5-10 минут
2. Дождитесь стабилизации температуры сенсора
3. Убедитесь, что видеопоток стабилен

### 4.4 Подготовка тестовых условий

#### 4.4.1 При использовании SMPTE Color Bars

1. Разместите тестовую таблицу SMPTE Color Bars на ровной поверхности
2. Убедитесь, что таблица равномерно освещена
3. Направьте камеру так, чтобы таблица занимала большую часть кадра
4. Проверьте отсутствие бликов и теней на таблице

#### 4.4.2 При отсутствии SMPTE Color Bars

При отсутствии тестовой таблицы используйте сцену со следующими элементами:

- Объекты различной яркости (белый лист, тёмные предметы)
- Объекты различных цветов (красный, зелёный, синий, жёлтый)
- Мелкие детали (текст, линии, узоры)
- Лицо человека (для оценки естественности цветов)

## 5. МЕТОДИКА ОЦЕНКИ ХАРАКТЕРИСТИК

### 5.1 ШУМНОСТЬ (Noise)

#### 5.1.1 Теоретическое обоснование

**Шум** — это случайные флуктуации яркости и цвета на изображении, возникающие в результате физических процессов в светочувствительном сенсоре камеры и усилительных каскадах электроники. Шум проявляется в виде зернистости, аналогичной той, что наблюдается на фотографиях, сделанных при плохом освещении.

Основные причины возникновения шума:

- **Тепловой шум** — хаотическое движение электронов в полупроводниковых элементах
- **С photonic noise** — статистические флуктуации фотонов
- **Считываемый шум** — шум АЦП (аналогово-цифрового преобразователя)
- **Фиксированный шум паттерна (Fixed Pattern Noise)** — структурный шум, характерный для конкретного экземпляра сенсора

Шум особенно заметен в тёмных областях изображения, где отношение сигнал/шум минимально.

#### 5.1.2 Визуальные признаки шума

Признаки повышенного уровня шума:

- Зернистость изображения, особенно в тёмных областях
- «Снег» — множество хаотично расположенных ярких точек
- Неравномерность однотонных областей
- Цветные пятна (хроматический шум)
- Использование камеры в условиях низкой освещённости усугубляет проблему

#### 5.1.3 Порядок проведения оценки

1. Подключите камеру и выведите видеопоток на монитор
2. Направьте камеру на сцену с преобладанием тёмных областей (например, угол помещения, тень)
3. Наблюдайте изображение в течение 30-60 секунд, не меняя положение камеры
4. Обратите внимание на наличие зернистости, особенно в областях средней и низкой яркости
5. Оцените степень влияния шума на детали изображения
6. При наличии используйте тестовую таблицу серого поля (70% серый) для более точной оценки

#### 5.1.4 Критерии оценки

| Результат | Визуальные признаки | Оценка |
|-----------|-------------------|--------|
| **ГОДЕН** | Минимальная или отсутствующая зернистость; изображение визуально чистое; детали чётко различимы в тёмных областях | ✓ |
| **НЕ ГОДЕН** | Выраженная зернистость, «снег» на изображении; зерна заметны невооружённым глазом; искажение деталей в тёмных областях | ✗ |

#### 5.1.5 Примечания

- Оценку проводите при нормальном освещении (300-500 люкс)
- При низкой освещённости шум естественно увеличивается — это нормальное поведение
- Оцените способность камеры работать в условиях низкой освещённости отдельно
- Обратите внимание на наличие цветного шума (красные, зелёные точки)

### 5.2 КОНТРАСТНОСТЬ (Contrast)

#### 5.2.1 Теоретическое обоснование

**Контрастность** (Contrast) — это разность между максимальной и мирогерностью яркости изображения, определяющая способность различать объекты в тенях и на свету. Контрастность является одной из ключевых характеристик, определяющих информативность видеоизображения.

Контрастность измеряется как отношение максимальной яркости к минимальной:

```
Контрастность = (Максимальная яркость - Минимальная яркость)
```

Практически, контрастность показывает, насколько «сочным» или «бледным» выглядит изображение.

Низкая контрастность приводит к:

- Затруднению различимости объектов в тенях
- «Бледному», малоконтрастному изображению
- Потере деталей в тёмных и светлых областях
- Общей неинформативности картинки

Причины низкой контрастности:

- Неправильная настройка диафрагмы
- Недостаточный динамический диапазон сенсора
- Некачественная оптика
- Неправильные настройки камеры (гамма-коррекция)

#### 5.2.2 Визуальные признаки низкой контрастности

- «Бледное, малоконтрастное изображение
- Детали в тенях неразличимы (сливаются в чёрное)
- Детали на светлых участках неразличимы (сливаются в белое)
- Общий серый оттенок изображения
- Малая разница между светлыми и тёмными областями

#### 5.2.3 Порядок проведения оценки

1. Направьте камеру на сцену с объектами различной яркости
2. В кадре должны присутствовать:
   - Очень светлые объекты (белый лист, источник света)
   - Очень тёмные объекты (чёрный предмет, тень)
   - Объекты средней яркости
3. Наблюдайте изображение
4. Определите, различимы ли детали в тенях (тёмных областях)
5. Определите, различимы ли детали на светлых участках
6. Оцените общую «сочность» и насыщенность изображения
7. Обратите внимание на уровень детализации в экстремальных условиях освещения

#### 5.2.4 Критерии оценки

| Результат | Визуальные признаки | Оценка |
|-----------|-------------------|--------|
| **ГОДЕН** | Хороший диапазон яркости; детали различимы в тенях и на свету; «сочное», насыщенное изображение; чёткое разграничение светлых и тёмных областей | ✓ |
| **НЕ ГОДЕН** | «Бледное» изображение; детали в тенях неразличимы; детали на светлых участках неразличимы; общий серый, малоконтрастный вид | ✗ |

#### 5.2.5 Примечания

- Контрастность можно оценить по перепадам яркости в кадре
- При наличии SMPTE оцените контрастность по переходу от белой полосы к чёрной
- Обратите внимание на гамма-кривую — она должна быть близка к линейной
- Проверьте, есть ли «завал» в тенях (потеря деталей в чёрном) и в светах (потеря деталей в белом)

### 5.3 РЕЗКОСТЬ (Sharpness)

#### 5.3.1 Теоретическое обоснование

**Резкость** (Sharpness) — это степень чёткости границ объектов и способность камеры передавать мелкие детали. Резкость является одной из наиболее важных характеристик, определяющих пригодность камеры для решения задач наблюдения и идентификации.

Резкость зависит от следующих факторов:

- **Разрешение сенсора** — количество пикселей на единицу площади
- **Качество объектива** — оптическое разрешение линз
- **Качество сенсора** — размер пикселя, технология
- **Настройки камеры** — фокусировка, аберрации
- **Сжатие** — степень компрессии видеопотока

Низкая резкость приводит к:

- Размытым границам объектов
- Невозможности различить мелкие детали
- Общей «замыленности» изображения
- Неспособности идентифицировать лица, номера

Резкость можно оценить по контурам объектов — чем чётче контур, тем выше резкость.

#### 5.3.2 Визуальные признаки низкой резкости

- Размытые границы объектов
- «Замыленное» изображение
- Нечёткие контуры предметов
- Мелкие детали неразличимы (мелкий текст, линии, узоры)
- Нечёткие края переходов цветов

#### 5.3.3 Порядок проведения оценки

1. Направьте камеру на объекты с мелкими деталями:
   - Текст (газета, книга, табличка)
   - Линии, провода, решётки
   - Узоры, мелкие предметы
   - Тестовая таблица с мелкими элементами
2. Оцените чёткость границ объектов
3. Определите, различимы ли мелкие детали
4. Обратите внимание на ровность контуров — нет ли «ореолов» вокруг предметов
5. При наличии используйте тестовую таблицу с мелкими элементами (шкалы)
6. Оцените способность камеры читать мелкий текст с расстояния

#### 5.3.4 Критерии оценки

| Результат | Визуальные признаки | Оценка |
|-----------|-------------------|--------|
| **ГОДЕН** | Чёткие границы объектов; мелкие детали различимы; контуры ровные, без ореолов; возможность прочитать мелкий текст; чёткое разграничение цветов | ✓ |
| **НЕ ГОДЕН** | Размытые границы; «замыленное» изображение; мелкие детали неразличимы; нечёткие контуры; невозможность прочитать мелкий текст | ✗ |

#### 5.3.5 Примечания

- Резкость — наиболее важная характеристика для задач идентификации
- Оцените способность камеры читать номер автомобиля с максимального расстояния
- Проверьте наличие артефактов резкости (ложные контуры, ореолы)
- Обратите внимание на фокусировку — камера должна быть правильно сфокусирована

---

### 5.4 ПЕРЕСВЕЧЕННОСТЬ 

#### 5.4.1 Теоретическое обоснование

**Пересвеченность** («пересвет») — это потеря деталей в ярких областях изображения вследствие избыточной экспозиции. Пересвеченность возникает, когда яркость некоторых участков изображения превышает максимально воспроизводимое значение, и эти участки отображаются как чисто белые (#FFFFFF в RGB) без какой-либо детализации.

Пересвеченность измеряется как процент площади кадра, на которой информация о деталях потеряна вследствие избыточной яркости.

Пересвеченность возникает по следующим причинам:

- **Неправильная экспозиция** — слишком большая выдержка или диафрагма
- **Недостаточный динамический диапазон** — сенсор не может воспроизвести полный диапазон яркостей
- **Яркие источники света в кадре** — солнце, лампы, отражения
- **Неправильные настройки** — слишком высокая яркость, усиление

#### 5.4.2 Визуальные признаки пересвеченности

- Белые («пересвеченные») области без деталей
- Потеря деталей в светлых участках
- «Выбитые» блики
- Белое пятно вместо источника света
- Отсутствие градаций в светлых областях

#### 5.4.3 Порядок проведения оценки

1. Направьте камеру на сцену с разнообразными уровнями яркости
2. Обратите внимание на наличие ярких объектов (источники света, отражения)
3. Определите, есть ли области, отображаемые как чисто белые без деталей
4. Оцените процент площади кадра с пересвеченными областями
5. Проверьте сохранность деталей в пересвеченных областях
6. При наличии яркого источника света в кадре оцените блики и ореолы

#### 5.4.4 Критерии оценки

| Результат | Визуальные признаки | Оценка |
|-----------|-------------------|--------|
| **ГОДЕН** | Отсутствие или минимальное количество пересвеченных областей (менее 5%); детали сохранены в подавляющем большинстве светлых участков; незначительные блики допустимы | ✓ |
| **НЕ ГОДЕН** | Значительные пересвеченные области (более 10-20%); потеря деталей в ярких участках; «выбитые» блики; белое пятно вместо источника света | ✗ |

#### 5.4.5 Примечания

- Оцените пересвеченность при наличии яркого источника света в кадре
- Обратите внимание на переходы яркости — должны быть плавными, без «обрезания»
- Проверьте наличие функции Auto Exposure (автоэкспозиция) и её работу
- Оцените динамический диапазон камеры — способность одновременно воспроизводить детали в тенях и на свету

### 5.5 ЦВЕТОПЕРЕДАЧА 

#### 5.5.1 Теоретическое обоснование

**Цветопередача** — это степень соответствия воспроизводимых цветов эталонным значениям. Цветопередача зависит от качества сенсора, обработки сигнала, цветовых фильтров и алгоритмов баланса белого.

Цветопередача особенно важна для:

- Систем распознавания лиц (естественность цветов кожи)
- Охраных систем (достоверность цветов одежды, автомобилей)
- Технического контроля (контроль качества продукции)
- Систем видеонаблюдения в особых условиях

Проблемы с цветопередачей могут быть вызваны:

- **Неправильным балансом белого** — смещение цветов
- **Низким качеством сенсора** — ограниченный цветовой охват
- **Плохой цветопередачей при низком освещении** — шум влияет на цвет
- **Неправильными настройками** — смещение цветового тона
- **Сильным сжатием** — потеря цветовой информации

#### 5.5.2 Эталонные цвета SMPTE Color Bars

При наличии тестовой таблицы SMPTE Color Bars используйте следующие эталонные значения:

| № | Цвет | Модель RGB | Описание |
|---|------|-----------|----------|
| 1 | Белый | R:255, G:255, B:255 | Максимальная яркость |
| 2 | Жёлтый | R:255, G:255, B:0 | Красный + зелёный |
| 3 | Голубой | R:0, G:255, B:255 | Зелёный + синий |
| 4 | Зелёный | R:0, G:255, B:0 | Чистый зелёный |
| 5 | Пурпурный | R:255, G:0, B:255 | Красный + синий |
| 6 | Красный | R:255, G:0, B:0 | Чистый красный |
| 7 | Синий | R:0, G:0, B:255 | Чистый синий |
| 8 | Чёрный | R:0, G:0, B:0 | Минимальная яркость |

#### 5.5.3 Визуальные признаки нарушения цветопередачи

- Неестественные оттенки цветов
- Смещение цветов (красный выглядит оранжевым, синий — фиолетовым)
- Низкая насыщенность цветов (бледные, «выцветшие» цвета)
- Высокая насыщенность (кислотные, неестественные цвета)
- Неправильный баланс белого (белый цвет имеет оттенок)

#### 5.5.4 Порядок проведения оценки (при наличии SMPTE)

1. Разместите тестовую таблицу SMPTE Color Bars в кадре
2. Убедитесь, что все 8 полос видны и равномерно освещены
3. Сравните цвета на изображении с эталонными значениями (см. п. 5.5.2)
4. Оцените:
   - Соответствие каждого цвета эталону
   - Насыщенность цветов
   - Отсутствие посторонних оттенков
5. Особое внимание уделите переходам между цветами

#### 5.5.5 Порядок проведения оценки (при отсутствии SMPTE)

При отсутствии тестовой таблицы используйте следующие референсные объекты:

1. **Кожа человека** — должна быть естественного оттенка (телесный цвет)
2. **Зелёная трава/листья** — должна быть естественно-зелёной (не жёлтой, не фиолетовой)
3. **Голубое небо** — должно быть естественно-голубым
4. **Белый лист бумаги** — должен быть белым (не серым, не жёлтым)
5. **Красные предметы** — должны быть красными (не оранжевыми, не розовыми)

Оцените естественность цветов этих объектов.

#### 5.5.6 Критерии оценки

| Результат | Визуальные признаки | Оценка |
|-----------|-------------------|--------|
| **ГОДЕН** | Цвета соответствуют эталонам или близки к ним; естественные оттенки; правильный баланс белого; отсутствие посторонних оттенков | ✓ |
| **НЕ ГОДЕН** | Значительные искажения цветов; неестественные оттенки; смещение цветов относительно эталонов; неправильный баланс белого | ✗ |

#### 5.5.7 Примечания

- Оцените баланс белого — белый цвет должен быть белым
- Проверьте цветопередачу при разных условиях освещения
- Обратите внимание на насыщенность — не слишком высокая, не слишком низкая
- При LOW освещённости цветопередача может ухудшаться — это нормально

## 11. ОСОБЕННОСТИ ОЦЕНКИ АНАЛОГОВЫХ КАМЕР

### 11.1 Общие положения

Аналоговые видеокамеры отличаются от IP-камер способом передачи видеосигнала. Вместо сетевого протокола IP аналоговые камеры передают видеосигнал в аналоговой форме по коаксиальному кабелю или витой паре. Несмотря на различия в технологии передачи данных, визуальная экспертная оценка качества изображения проводится аналогично IP-камерам, с учётом особенностей, описанных в настоящем разделе.

### 11.2 Типы аналоговых камер

#### 11.2.1 Классические аналоговые камеры (CVBS)

Стандартный аналоговый видеосигнал (Composite Video Baseband Signal). Разрешение:

- PAL: 720×576 пикселей (D1)
- NTSC: 720×480 пикселей

#### 11.2.2 HD-аналоговые камеры (AHD/TVI/CVI)

Современные HD-стандарты аналоговой передачи высокой чёткости:

| Стандарт | Разрешение | Описание |
|----------|------------|----------|
| AHD | 1280×720, 1920×1080 | Analog High Definition |
| TVI | 1280×720, 1920×1080 | Transport Video Interface |
| CVI | 1280×720, 1920×1080 | Composite Video Interface |
| SDI | 1920×1080 | Serial Digital Interface (цифровой по коаксиалу) |

### 11.3 Отличия в методике оценки

#### 11.3.1 Подключение оборудования

При оценке аналоговых камер используется следующая схема подключения:

```
┌─────────────────────────────────────────────────────────────────┐

│                     аналоговой камеры                            │

│                                                                 │

│   │  Аналог.   │────────▶│   BNC/      │──────▶│ DVR или  │    │
│   │   камера   │  video  │   кабель    │       │  монитор │    │
│   │            │   CVBS  │             │       │ (вход    │    │

│   └─────────────┘         └─────────────┘       └──────────┘    │
│         │                                                   │
│         ▼                                                   │

```

#### 11.3.2 Оборудование для подключения

| Оборудование | Требования | Назначение |
|--------------|-------------|------------|
| Аналоговая камера | Объект оценки | Испытываемое оборудование |
| Видеорегистратор (DVR) или монитор | BNC-вход | Приём аналогового сигнала |
| Коаксиальный кабель | RG59 или RG6 | Соединение камеры с DVR |
| Блок питания | 12В DC / 24В AC | Питание камеры |
| Штатив | Стандартное крепление | Фиксация камеры |

#### 11.3.3 Настройка камеры

1. Подключите камеру к блоку питания (12В или 24В , согласно спецификации)
2. Подключите видеовыход камеры к видеорегистратору или монитору через BNC-разъём
3. Установите переключатель режимов (если есть) в режим HD (AHD/TVI/CVI) для камер высокой чёткости
4. Настройте разрешение (если доступно): 1080p или 720p
5. Установите частоту кадров: PAL (25 fps) или NTSC (30 fps)
6. Отключите функции обработки (шумоподавление, WDR) для чистоты оценки

### 11.4 Особенности оценки характеристик

#### 11.4.1 Шумность (Noise)

**Специфика аналоговых камер:**

Аналоговые камеры подвержены дополнительным видам помех, связанным с аналоговой передачей сигнала:

- **Наводки** — помехи от электросети и электрооборудования
- **Полосы помех** — горизонтальные полосы при проблемах с заземлением
- **Мерцание** — нестабильность сигнала при плохом соединении
- **Снег»** — аналогично IP-камерам, но может усиливаться помехами

**Дополнительные признаки проблем:**

| Признак | Возможная причина |
|---------|------------------|
| Горизонтальные полосы | Наводка 50/60 Гц, проблема с заземлением |
| Мерцание изображения | Плохой контакт в разъёме, нестабильное питание |
| «Снег» и рябь | Плохое качество кабеля, слишком длинная линия |
| Помехи «ёлочкой» | Несовместимость стандартов PAL/NTSC |

**Критерии оценки:**

| Результат | Визуальные признаки | Оценка |
|-----------|-------------------|--------|
| **ГОДЕН** | Минимальная зернистость; отсутствие помех; стабильное изображение | ✓ |
| **НЕ ГОДЕН** | Выраженная зернистость; наводки; мерцание; статические помехи | ✗ |

#### 11.4.2 Контрастность 

Контрастность аналоговых камер оценивается аналогично IP-камерам. Однако следует учитывать:

- **Динамический диапазон** аналоговых камер, как правило, ниже, чем у IP-камер
- **Эффект «плавания»** — возможные искажения контраста при длинных кабелях
- **Потеря сигнала** на длинных линиях может влиять на контрастность

**Критерии оценки:**

| Результат | Визуальные признаки | Оценка |
|-----------|-------------------|--------|
| **ГОДЕН** | Хороший диапазон яркости; детали различимы в тенях и на свету | ✓ |
| **НЕ ГОДЕН** | «Плоское» изображение; детали неразличимы | ✗ |

#### 11.4.3 Резкость

При оценке резкости аналоговых камер учитывайте:

- **Разрешение** ограничено стандартом (PAL: 720×576, NTSC: 720×480, HD: 1280×720 или 1920×1080)
- **Эффект затухания** — потеря резкости на длинных кабелях
- **Артефакты** — возможные искажения, связанные с преобразованием сигнала

**При оценке HD-аналоговых камер (AHD/TVI/CVI):**

При использовании камер HD-формата разрешение сопоставимо с IP-камерами 720p/1080p. Оценка проводится по аналогичным критериям.

**Критерии оценки:**

| Результат | Визуальные признаки | Оценка |
|-----------|-------------------|--------|
| **ГОДЕН** | Чёткие границы объектов; мелкие детали различимы для данного разрешения | ✓ |
| **НЕ ГОДЕН** | Размытые границы; для HD-камер — значительное размытие | ✗ |

#### 11.4.4 Пересвеченность 

Пересвеченность аналоговых камер оценивается аналогично IP-камерам. Дополнительно учитывайте:

- **Диапазон регулировки** — аналоговые камеры могут иметь ограниченные возможности настройки экспозиции
- **Функция Auto Iris** — при наличии автоматической диафрагмы проверьте её работу

**Критерии оценки:**

| Результат | Визуальные признаки | Оценка |
|-----------|-------------------|--------|
| **ГОДЕН** | Отсутствие или минимальные пересвеченные области; детали сохранены | ✓ |
| **НЕ ГОДЕН** | Значительные пересвеченные области; потеря деталей | ✗ |

#### 11.4.5 Цветопередача 

Цветопередача аналоговых камер имеет следующие особенности:

- **Ограниченный цветовой охват** — особенно у старых CVBS-камер
- **Возможное смещение цветов** — из-за особенностей аналоговой передачи
- **Наводки на цвет** — помехи могут влиять на цветовую составляющую

**Критерии оценки:**

| Результат | Визуальные признаки | Оценка |
|-----------|-------------------|--------|
| **ГОДЕН** | Цвета близки к естественным; правильный баланс белого | ✓ |
| **НЕ ГОДЕН** | Значительные искажения цветов; неестественные оттенки | ✗ |

### 11.5 Дополнительные проверки для аналоговых камер

#### 11.5.1 Проверка совместимости стандартов

#### 11.5.2 Проверка линии передачи

При использовании длинных кабелей (>100 м) проверьте:

- Отсутствие изломов кабеля
- Заземление экрана кабеля
- Отсутствие источников помех вблизи кабеля

## 12. ПРОВЕРКА ИК-ПОДСВЕТКИ IP И АНАЛОГОВЫХ КАМЕР

### 12.1 Общие положения

Инфракрасная (ИК) подсветка используется для обеспечения видеонаблюдения в условиях низкой освещённости или полной темноты. ИК-подсветка состоит из массива инфракрасных светодиодов, излучающих свет с длиной волны 850 нм или 940 нм.

### 12.2 Метод проверки с использованием затемняющего светофильтра

Данный метод является наиболее простым и эффективным способом проверки работоспособности ИК-подсветки без специального оборудования.

#### 12.2.1 Сущность метода

Затемняющий светофильтр (или поляризационный фильтр переменной плотности) прикладывается к объективу камеры. Фильтр значительно снижает количество видимого света, попадающего на сенсор камеры. Камера воспринимает это как резкое снижение освещённости и автоматически включает ИК-подсветку. Через тот же фильтр можно визуально проверить свечение всех ИК-светодиодов.

**Типы фильтров, пригодных для проверки:**

| Фильтр | Описание |
|--------|----------|
| Затемняющий ND-фильтр | Переменной плотности (ND8-ND64) |
| Поляризационный фильтр | С регулируемой поляризацией |

#### 12.2.2 Оборудование

| Оборудование | Назначение |
|--------------|------------|
| Затемняющий светофильтр | Создание условий низкой освещённости |
| Монитор | Отображение видеосигнала |
| Штатив | Фиксация камеры |

#### 12.2.3 Подготовка к проверке

1. **Установите камеру** на штатив, направьте на тестовую сцену или монитор
2. **Подключите камеру** к монитору или DVR
3. **Расположитесь** так, чтобы видеть монитор и иметь доступ к объективу камеры
4. **Подготовьте фильтр** — убедитесь, что он достаточно тёмный

#### 12.2.4 Порядок проведения проверки

**Этап 1: Проверка в нормальных условиях**

1. Включите камеру и монитор
2. Убедитесь, что изображение нормальное при стандартном освещении
3. Зафиксируйте исходное состояние

**Этап 2: Применение затемняющего фильтра**

1. **Приложите фильтр** непосредственно к объективу камеры (или как можно ближе к нему)
2. **Убедитесь**, что фильтр закрывает весь объектив
3. **Выждите 2-5 секунд** — камера должна среагировать на снижение освещённости

**Этап 3: Наблюдение за включением ИК-подсветки**

1. **Следите за монитором** — изображение должно измениться (перейти в ночной режим)
2. **Оцените качество изображения** — должно быть видно ИК-подсветку
3. **Проверьте визуально** — если возможно, посмотрите на камеру (для 850 нм видно красное свечение)

**Этап 4: Проверка всех светодиодов ИК-подсветки**

1. **Наблюдайте за светодиодами** на корпусе камеры (или через фильтр)
2. **Подсчитайте** количество горящих диодов
3. **Проверьте равномерность** — все диоды должны гореть
4. **Обратите внимание** на неравномерное свечение или нерабочие диоды

**Этап 5: Снятие фильтра**

1. **Снимите фильтр** с объектива
2. **Дождитесь** выключения ИК-подсветки (автоматическое переключение)
3. **Проверьте** возврат в дневной режим

### 12.3 Критерии оценки ИК-подсветки

| Результат | Визуальные признаки | Оценка |
|-----------|-------------------|--------|
| **ГОДЕН** | ИК-подсветка включается автоматически при затемнении; все светодиоды горят равномерно; изображение в ИК-режиме достаточного качества | ✓ |
| **НЕ ГОДЕН** | ИК-подсветка не включается; часть светодиодов не горит; неравномерное свечение; изображение недостаточного качества | ✗ |

### 12.4 Дополнительные проверки

#### 12.4.1 Проверка дальности ИК-подсветки

При включённой ИК-подсветке:

1. Направьте камеру на объект
2. Увеличивайте расстояние до объекта
3. Зафиксируйте расстояние, при котором изображение остаётся различимым

Ориентировочные значения:

| Тип камеры | Дальность ИК-подсветки |
|-----------|----------------------|
| Внутренняя (cube) | 5-10 метров |
| Купольная | 10-20 метров |
| Цилиндрическая | 20-40 метров |
| Уличная | 30-100 метров |

#### 12.4.2 Проверка равномерности освещения

При ИК-подсветке в темноте:

1. Направьте камеру на белую поверхность
2. Оцените равномерность освещения по всему кадру
3. Проверьте отсутствие ярких пятен и затемнённых зон

| Результат | Признак |
|-----------|---------|
| **ГОДЕН** | Равномерное освещение по всей площади |
| **НЕ ГОДЕН** | Яркие пятна, неравномерное освещение |

#### 12.4.3 Проверка переключения режимов

1. **Автоматический режим:** При затемнении — включение, при освещении — выключение
2. **Ручной режим:** Проверьте переключение в веб-интерфейсе (IP-камеры) или настройках (DVR для аналоговых)

### 12.5 Регистрация результатов

| Характеристика | Результат | Комментарий |
|----------------|-----------|-------------|
| Включение при затемнении | ГОДЕН / НЕ ГОДЕН | |
| Работа всех светодиодов | ГОДЕН / НЕ ГОДЕН | |
| Равномерность подсветки | ГОДЕН / НЕ ГОДЕН | |
| Переключение режимов | ГОДЕН / НЕ ГОДЕН | |
| **ИТОГ (ИК-подсветка)** | **ГОДЕН / НЕ ГОДЕН** | |

### 12.6 Примечания

1. Затемняющий фильтр должен быть достаточно тёмным (ND8 или больше)
2. Для камер 940 нм свечение диодов практически не видно — проверяйте по изображению на мониторе
3. Время срабатывания ИК-подсветки: обычно 2-5 секунд
4. При проверке учитывайте, что ИК-диоды деградируют со временем
