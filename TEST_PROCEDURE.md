## 1. ОБЩИЕ ПОЛОЖЕНИЯ

### 1.1 Назначение документа

Настоящая «Программа и методика испытаний» (далее — ПМИ) устанавливает порядок проведения испытаний IP-видеокамер с использованием автоматизированной системы «IP Camera Quality Analyzer» для оценки качества видеоизображения.

### 1.2 Область применения

ПМИ применяется при:
- Приёмочных испытаниях вновь устанавливаемых IP-камер
- Периодическом контроле качества видеоизображения
- Сравнительном анализе различных моделей камер
- Настройке и оптимизации параметров видеокамер
- Сертификационных испытаниях видеооборудования

### 1.3 Нормативные ссылки

- ГОСТ Р 51558-2014 «Средства охранной видеотехники»
- ГОСТ 34254-2017 «Камеры видеокамер цифровые»
- IEEE 802.11 — Стандарты беспроводных сетей
- RFC 2327 — RTSP (Real Time Streaming Protocol)

### 1.4 Термины и определения

| Термин | Определение |
|--------|-------------|
| **IP-видеокамера** | Цифровая видеокамера, передающая видеоданные по протоколу IP |
| **RTSP** | Протокол потоковой передачи данных в реальном времени |
| **Качество изображения** | Совокупность характеристик видеоизображения, определяющих его пригодность для решения задач наблюдения |
| **SMPTE Color Bars** | Тестовая таблица для проверки цветопередачи видеооборудования |
| **Метрика качества** | Количественный показатель, характеризующий отдельный аспект качества изображения |

## 2. ЦЕЛИ И ЗАДАЧИ ИСПЫТАНИЙ

### 2.1 Цели испытаний

Основной целью испытаний является определение качественных характеристик видеоизображения IP-видеокамер с использованием автоматизированных методов анализа.

### 2.2 Задачи испытаний

1. **Измерение шумовых характеристик**
   - Определение уровня цифрового шума в изображении
   - Оценка влияния шума на информативность видео

2. **Анализ контрастности**
   - Измерение динамического диапазона яркости
   - Определение различимости объектов в тенях и на свету

3. **Оценка резкости изображения**
   - Измерение четкости границ объектов
   - Определение способности камеры передавать мелкие детали

4. **Анализ экспозиции**
   - Выявление пересвеченных и недосвеченных областей
   - Оценка правильности настройки экспозиции

5. **Проверка цветопередачи**
   - Обнаружение тестовой таблицы SMPTE Color Bars
   - Измерение точности воспроизведения цветов

6. **Определение комплексной оценки**
   - Расчёт обобщённого показателя качества
   - Классификация камеры по категориям качества

## 3. ОБЪЕКТ ИСПЫТАНИЙ

### 3.1 Перечень испытываемого оборудования

| № | Наименование | Количество | Примечание |
|---|-------------|------------|------------|
| 1 | IP-видеокамера (объект испытаний) | 1 шт. | Поставляется заказчиком |
| 2 | Сервер/ПК с ПО «IP Camera Quality Analyzer» | 1 шт. | Испытательное оборудование |
| 3 | Кабель Ethernet | 1 шт. | Category 5e или выше |
| 4 | Тестовая таблица SMPTE Color Bars | 1 шт. | А3 или больше |
| 5 | Источник освещения (для тестовых условий) | 1 шт. | Регулируемый |

### 3.2 Требования к испытываемой камере

Камера должна поддерживать:
- Протокол RTSP (Real Time Streaming Protocol)
- Разрешение не менее 1920×1080 (Full HD)
- Кодек H.264 или H.265
- Стабильное сетевое подключение

### 3.3 Требования к программному обеспечению

Для проведения испытаний используется ПО «IP Camera Quality Analyzer» версии 0.0.1.

## 5. ПОДГОТОВКА К ИСПЫТАНИЯМ

### 5.1 Подготовка испытательного стенда

#### Шаг 1: Сборка оборудования

1. Разместите камеру на штативе или кронштейне
2. Направьте камеру на тестовую таблицу SMPTE Color Bars
3. Убедитесь, что таблица полностью помещается в кадре
4. Подключите камеру к сети Ethernet
5. Подключите ПК к той же сети

```
┌─────────────────────────────────────────────────────────────────┐
│                     Схема стенда                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────┐         ┌─────────────┐       ┌──────────┐  │
│   │   Камера    │────────▶│    Сеть     │──────▶│    ПК    │  │
│   │  (объект    │  RTSP   │  Ethernet   │       │ (анализ) │  │
│   │ испытаний)  │         │             │       │          │  │
│   └─────────────┘         └─────────────┘       └──────────┘  │
│         │                                                   │
│         ▼                                                   │
│   ┌───────────────┐                                       │
│   │   SMPTE       │                                       │
│   │   таблица     │                                       │
│   └───────────────┘                                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### Шаг 2: Настройка освещения

1. Включите источник освещения
2. Установите освещённость в соответствии с требуемыми условиями:
   - **Нормальные условия:** 300-500 люкс
   - **Высокая освещённость:** 1000+ люкс
   - **Низкая освещённость:** 50-100 люкс

#### Шаг 3: Настройка камеры

1. Войдите в веб-интерфейс камеры
2. Установите следующие параметры:
   - Разрешение: 1920×1080 (или максимальное)
   - Частота кадров: 25-30 fps
   - Битрейт: 2048-4096 кбит/с
   - Кодек: H.264
3. Отключите функции сжатия, ухудшающие качество
4. Отключите WDR (если не требуется для теста)

#### Шаг 4: Настройка ПО

1. Запустите приложение «IP Camera Quality Analyzer»
2. Убедитесь, что все компоненты загружены корректно

### 5.2 Подготовка к проведению измерений

1. Дайте камере прогреться в течение 5-10 минут
2. Убедитесь, что видеопоток стабилен
3. Проверьте отсутствие помех в сети
4. Подготовьте форму протокола испытаний

## 7. МЕТОДИКА ОЦЕНКИ КАЧЕСТВА

### 7.1 Метрики качества

Система оценивает качество изображения по пяти метрикам:

#### 7.1.1 Шумность (Noise) — Вес 20%

**Что измеряется:** Уровень цифрового шума

**Метод измерения:**
1. Применяется гауссово размытие к изображению
2. Вычисляется разница между оригиналом и размытой версией
3. Рассчитывается дисперсия разницы

**Формула:**
```
NoiseScore = 100 - (variance(original - blurred) / MAX_NOISE_VARIANCE) × 100
```

**Интерпретация:**
| Диапазон | Оценка | Примечание |
|----------|--------|------------|
| 80-100 | Отличное | Минимальный шум |
| 60-79 | Хорошее | Незначительный шум |
| 40-59 | Удовлетворительное | Умеренный шум |
| 0-39 | Плохое | Высокий уровень шума |

#### 7.1.2 Контрастность (Contrast) — Вес 20%

**Что измеряется:** Диапазон яркости изображения

**Метод измерения:**
1. Находится минимальное и максимальное значение яркости
2. Вычисляется разница max - min

**Формула:**
```
ContrastScore = min(100, (actualContrast / IDEAL_CONTRAST) × 100)
IDEAL_CONTRAST = 160
```

**Интерпретация:**
| Диапазон | Оценка | Примечание |
|----------|--------|------------|
| 80-100 | Отличный | 160-180 |
| 60-79 | Хороший | 120-159 |
| 40-59 | Удовлетворительный | 80-119 |
| 0-39 | Плохой | < 80 |

#### 7.1.3 Резкость (Sharpness) — Вес 30%

**Что измеряется:** Чёткость границ и деталей

**Метод измерения:**
1. Применяется оператор Лапласа
2. Вычисляется дисперсия результата

**Формула:**
```
SharpnessScore = min(100, (laplacianVariance / IDEAL_SHARPNESS) × 100)
IDEAL_SHARPNESS = 400
```

**Интерпретация:**
| Диапазон | Оценка | Примечание |
|----------|--------|------------|
| 80-100 | Отличная | Чёткие детали |
| 60-79 | Хорошая | Незначительное размытие |
| 40-59 | Удовлетворительная | Видимое размытие |
| 0-39 | Плохая | Сильное размытие |

#### 7.1.4 Пересвеченность (Overexposed) — Вес 10%

**Что измеряется:** Процент областей с потерей деталей

**Метод измерения:**
1. Подсчитываются пиксели с яркостью > 245
2. Вычисляется процент от общего числа

**Формула:**
```
OverexposedPercent = (pixels > 245 / totalPixels) × 100
```

**Интерпретация:**
| Диапазон | Оценка | Примечание |
|----------|--------|------------|
| 80-100 | Нет пересвета | < 5% |
| 60-79 | Незначительный | 5-10% |
| 40-59 | Умеренный | 10-20% |
| 0-39 | Сильный | > 20% |

#### 7.1.5 SMPTE Color Bars — Вес 20%

**Что измеряется:** Точность цветопередачи

**Метод измерения:**
1. Обнаружение SMPTE таблицы в кадре
2. Измерение цвета каждой из 8 полос
3. Сравнение с эталонными значениями

**Эталонные цвета SMPTE:**
| № | Цвет | RGB |
|---|------|-----|
| 1 | Белый | 255, 255, 255 |
| 2 | Жёлтый | 255, 255, 0 |
| 3 | Голубой | 0, 255, 255 |
| 4 | Зелёный | 0, 255, 0 |
| 5 | Пурпурный | 255, 0, 255 |
| 6 | Красный | 255, 0, 0 |
| 7 | Синий | 0, 0, 255 |
| 8 | Чёрный | 0, 0, 0 |

**Интерпретация:**
| Диапазон | Оценка | Примечание |
|----------|--------|------------|
| 80-100 | Отличная | Цвета соответствуют |
| 60-79 | Хорошая | Незначительные отклонения |
| 40-59 | Средняя | Заметные отклонения |
| 0-39 | Плохая | Серьёзные отклонения |

### 7.2 Расчёт комплексной оценки

**Формула расчёта:**

```
OverallScore = Noise×0.20 + Contrast×0.20 + Sharpness×0.30 + Overexposed×0.10 + SMPTE×0.20
```

**Пример расчёта:**

| Метрика | Значение | Вес | Вклад |
|---------|----------|-----|-------|
| Шум | 85 | 0.20 | 17.0 |
| Контраст | 80 | 0.20 | 16.0 |
| Резкость | 90 | 0.30 | 27.0 |
| Пересвет | 90 | 0.10 | 9.0 |
| SMPTE | 85 | 0.20 | 17.0 |
| **Итого** | | | **86.0** |

## 8. ОБРАБОТКА РЕЗУЛЬТАТОВ

### 8.1 Сбор данных

1. Автоматический сбор метрик в течение периода наблюдения
2. Регистрация минимальных, максимальных и средних значений
3. Фиксация времени и условий проведения измерений

### 8.2 Статистическая обработка

Для каждой метрики рассчитываются:
- Среднее арифметическое значение (Mean)
- Среднеквадратическое отклонение (StdDev)
- Минимальное значение (Min)
- Максимальное значение (Max)

### 8.3 Определение комплексной оценки

1. Используются средние значения метрик за период наблюдения
2. Применяются весовые коэффициенты
3. Рассчитывается итоговая оценка

| SMPTE не обнаружена | Таблица не в кадре | Направьте камеру на таблицу |
| | Плохое освещение | Улучшите освещение |
